  O QUE É A CULTURA UNDERHOLE?
  ============================
  
  Somos uma Cibercultura que despreza todo tipo de complexidade, ela é descartada
  na ráiz.
  Nós utilizamos o SFTP como protocolo padrão, uma alternativa segura nativamente
  ao padronizado HTTP(s).
  Por não termos uma indexação, que vasculha os servidores como o Google, você é
  livre para divulgar somente para quem você quiser.
  Temos um servidor o primeiro que hospeda o DAD HOLE. É um arquivo em texto plano
  que divulga os Holes. Está 'Indexação' é feita somente para quem o quiser.
  
  Hole? O que é?
  Hole é o termo que usamos para o espaço de cada usuário. Este buraco é aonde
  você hospeda os seus conteúdos abertamente para todos. É um website? Não.
  Todo conteúdo é formado por arquivos selvagens como .txt. e diretórios. Você
  pode hospedar qualquer conteúdo, como imagens, vídeos entre outras coisas. Mas
  o seu servidor, principalmente caseiro será comprometido pela banda que ira
  consumir.
  A ideia principal do UnderHole é hospedar arquivos de texto plano pelo baixo
  consumo de banda já que um mega arquivo em Texto crú não passará de alguns Kbytes.
  Um arquivo em texto plano de 15k de linhas tem em média 560 Kbytes e não passará
  por nenhuma renderização :)
  
  A banda/recurso usado em um servidor é minímo mesmo com um alto número de acessos.
  Em nosso mundo ainda existem pessoas que usam internet discada, como seu principal
  acesso a WEB. Elas serão as mais beneficiadas, os paises mais pobres se beneficiam
  e agradecem.
    
  Nessa forma medieval criamos um HoleSpace. Onde é totalmente descentralizado.
  Você pode inclusive copiar facilmente o Hole de outra pessoa e passar a hospedar
  como um espelho.
  Isso é digno quando o conteúdo para você é realmente útil.
  Você também pode criar no mesmo servidor outros Holes, como por exemplo para um
  amigo(s) de confiança, permitindo assim acesso de 'Escrita/Leitura' naquele
  determinado diretório.
  Você e seu(s) amigo(s) terão um custo quase minimo hospedando em grupo.
    
  
  VOCÊ NÃO PRECISA.
  =================
  
   * Para acesso você não precisará ter um browser! E sim um simples cliente SFTP
   Hoje em dia até gerenciadores como 'Thunar' 'Nautilus' Entre outros tem suporte
   nativo ao acesso SFTP.
   Muitos gerenciados de celular nativos também te oferecem este recurso.
      
   * Você não precisará comprar um dominio! O seu dominio é o ip do seu servidor.
   Para servidores caseiros como alternativa você pode usar algum tipo de DNS.
   Não recomendamos você utilizar um dominio, invista seu dinheiro em outra coisa.
   
   * Você NÃO precisará comprar um certificado SSL. O próprio protocolo SSH já
   disponibiliza a criptografia nativamente. Ou seja todo transporte é seguro.
   
   * Se você usa alguma distribuição Linux você pode criar alias/nicks facilmente
   para seus servidores favoritos em /etc/hosts. Assim permitindo acesso mais fácil.
   
   * Autenticação não depende de chaves cruzadas, o acesso a Hole é direto especifi-
   cando somente o usuário que por padrão utilizamos 'anonhole'.
   Mas você pode querer criar um hole privado e dar acesso somente com chaves cru-
   zadas.
   
   
   COMO FUNCIONA A CONEXÃO?
   ========================
   
   É direta! Você tem acesso diretamente ao servidor usando como intermédio o SFTP.
   Chamamos esse tipo de conexão de 'GET.AND.EXECUTE' ou 'GAE'. Nesse formato é
   direto sem necessitar nada além disso como um renderização.
   


      +---+        G A E       +----+
      |   |    Prótocolo SSH   |    |
      |   | +----------------> |    |
      +---+                    +----+
      Usuário                   Hole
  
   No Hole do usuário, ele poderá dar permissões aos seus visitantes como desejar.
   O acesso SSH não é fornecido e não recomendamos isso.
